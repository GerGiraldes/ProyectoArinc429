De que Trata Nuestro Proyecto

¿De qué trata este proyecto?

El sistema está compuesto por varios módulos conectados entre sí:

  1. Interfaz Web: El usuario ingresa una dirección deseada (por ejemplo: 270° Oeste) directamente desde una página web visual.
  2. Servidor: Esa información es enviada a un servidor local, que la procesa y la prepara para transmitirla.
  3. Arduino + Transceptor ARINC 429: Un microcontrolador Arduino recibe la dirección desde el servidor, la codifica según el protocolo ARINC 429 y la transmite mediante un transceptor especializado.
  4. Instrumento Real: Finalmente, un instrumento físico (una brújula electrónica real de avión) recibe esta señal y cambia su lectura según el valor ingresado desde la web.

¿Qué es ARINC 429?

ARINC 429 es un protocolo de comunicación utilizado en aeronaves reales. Se encarga de transmitir información crítica entre instrumentos de vuelo, garantizando precisión, velocidad y confiabilidad. Este proyecto lo utiliza para simular una integración casi real de una cabina de avión.

¿Por qué es interesante?

Este proyecto demuestra cómo una tecnología usada en la aviación real puede ser simulada, comprendida y controlada desde herramientas simples como una página web. Es una prueba clara de cómo el mundo digital se conecta con el mundo físico.

Además, permite:

  • Visualizar cómo funcionan los sistemas embarcados en aeronaves.
  • Interactuar de forma remota con instrumentos reales.
  • Aprender sobre codificación, protocolos de comunicación y sistemas distribuidos.

Posibilidades Futuras

Este concepto se puede ampliar fácilmente para simular más instrumentos como el altímetro, velocímetro o horizonte artificial. Incluso se podría integrar a un simulador de vuelo completo, con retroalimentación visual y física.
